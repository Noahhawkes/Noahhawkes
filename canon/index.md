# Canon Index

**Status:** Operational canon — v0.1

**Purpose:** Front door and routing table for the oracle-ai-core continuity architecture.

This index exists so a future human reader, AI system, crawler, or family member can distinguish operational files from unresolved drafts and experimental material.

## Status taxonomy

Every major file in this repository should declare one of the following statuses near the top of the file.

### Operational canon

Meaning: usable as current doctrine, protocol, or routing authority.

Use when the file defines the current working rule set.

Operational canon may still be revised, but it is safe to use as the current reference.

### Active protocol

Meaning: usable as an operating procedure.

Use when the file defines steps a system or person should execute.

### Active backlog

Meaning: known work, risks, gaps, or engineering tasks that are not yet resolved.

Use when the file tracks what remains broken or incomplete.

### Open draft

Meaning: not settled. Do not treat as doctrine.

Use when the file contains a proposed concept that has not survived review or testing.

### Unresolved target

Meaning: named goal, unsolved problem, or future artifact.

Use when the file names what must eventually exist but does not claim completion.

### Experimental

Meaning: speculative, test-only, or non-authoritative.

Use when the file is useful for exploration but must not govern reconstruction.

### Deprecated

Meaning: superseded, corrected, or no longer authoritative.

Use when a file remains for history but must not be used as current doctrine.

## Authority warning

Status is not the same as truth authority.

This index is the best place to start reading, but it does not outrank stronger primary evidence.

Truth order remains:

1. Signed, timestamped, or hash-verifiable primary records
2. Published papers, website pages, and dated public artifacts
3. Current public canon in this repository
4. Dated private documents and drafts
5. Raw transcripts
6. Model memory
7. Model synthesis

Orientation order tells a reader where to start.

Truth order tells a reader what wins when sources conflict.

## Unresolved targets

### True MVIK, Minimum Viable Identity Kernel

Status: Unresolved target

The true identity kernel is not completed.

The current `canon/mvik.md` file was reclassified as MVDK because it contains governance and discrimination rules, not positive identity content.

A true MVIK would need to address reasoning patterns, value hierarchy, narrative cadence, characteristic humor, autobiographical anchors, decision heuristics, and emotional friction without creating a style-complete, judgment-empty impersonation surface.

### Family-readable witness layer

Status: Unresolved target

The family-readable layer does not yet exist.

This layer must allow family members to recognize Noah without parsing dense technical doctrine.

It should preserve ordinary stories, humor, decisions, values, and lived context in plain language.

### Judgment corpus

Status: Unresolved target

The judgment corpus does not yet exist.

It should record documented decisions with context, available options, rejected alternatives, reasons, constraints, and provenance.

This is required for judgment-drift testing.

### Automated validation logic

Status: Unresolved target

The discriminator is currently mostly textual.

Future work should add deterministic checks where possible: schema validation, required citations, concrete exclusion matching, unsupported precision flags, and contradiction detection.
